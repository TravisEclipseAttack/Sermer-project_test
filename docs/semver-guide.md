Руководство по SemVer в этом проекте

Правила версионирования

MAJOR версия (X.0.0)
- Несовместимые изменения API
- Удаление устаревшего функционала
- Пример: v2.0.0

MINOR версия (0.X.0) 
- Новая обратно-совместимая функциональность
- Добавление новых методов/классов
- Пример: v1.3.0

PATCH версия (0.0.X)
- Исправления ошибок
- Оптимизация без изменения API
- Пример: v1.2.1

Процесс разработки

1. Для новой фичи: `git checkout -b feature/name`
2. Для исправления: `git checkout -b fix/name`
3. Для релиза: `git checkout -b release/v1.0.0`
